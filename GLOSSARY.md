# 用語集（Swift / SwiftUI / Xcode 初心者向け）

このプロジェクトを読み進めるために知っておくと良い用語を、わかりやすい意味でまとめたものです。

## Swift とは

**Apple が開発したプログラミング言語**です。「iOS 用の言語」ではなく、**Apple のプラットフォーム全般で使う言語**です。

| プラットフォーム | 用途 |
|---|---|
| iOS | iPhone アプリ |
| macOS | Mac アプリ（← この Forensic プロジェクトはコレ） |
| iPadOS | iPad アプリ |
| watchOS | Apple Watch アプリ |
| tvOS | Apple TV アプリ |
| サーバーサイド | Vapor 等のフレームワークで Linux サーバーでも動く |

### 混同しやすい用語の整理

| 用語 | 正体 |
|---|---|
| Swift | プログラミング言語そのもの（文法） |
| SwiftUI | Swift で **UI（画面）を作るためのフレームワーク**。iOS でも macOS でも共通で使える |
| UIKit | 古くからある iOS/iPadOS 専用の UI フレームワーク（SwiftUI 以前の主流） |
| AppKit | 古くからある macOS 専用の UI フレームワーク |
| Xcode | Swift でコードを書いたりビルドしたりする**開発ツール（アプリ）** |

このプロジェクトは「Swift 言語 + SwiftUI フレームワーク」で作られた **macOS アプリ**です（iOS アプリではありません）。

## Swift 言語の用語

| 用語 | わかりやすい意味 |
|---|---|
| `var` / `let` | `var` は後で変更できる箱、`let` は一度入れたら変えられない箱 |
| Optional（`?`） | 「値があるかもしれないし、ない(nil)かもしれない」を表す型。`String?` なら文字列か nil |
| nil | 「何も入っていない」状態。他言語の null と同じ |
| `if let` / `guard let` | Optional の中身を安全に取り出すための書き方（nil なら処理しない） |
| struct | データのまとまりを表す型。Swift では基本これを使う（コピーされる＝値型） |
| class | struct と似ているが、コピーされず共有される（参照型）。`@Observable` なオブジェクトなどで使う |
| protocol | 「これができること」を定義する契約書。例：`View` プロトコルは「body を持つこと」を要求する |
| extension | 既存の型に後から機能を追加する仕組み |
| enum | 決まった選択肢の中から1つを表す型（例：`.success` か `.failure` か） |
| closure（クロージャ） | `{ }` で書ける「その場で作る小さな関数」。ボタンを押した時の処理などによく使う |
| async/await | 時間がかかる処理（ファイル読み込みなど）を「待つ」ための書き方。UI が固まらないようにする |

## SwiftUI の用語

| 用語 | わかりやすい意味 |
|---|---|
| View | 画面の部品。ボタンもテキストも画面全体もすべて「View」 |
| body | その View が「実際にどう見えるか」を返す場所（必須プロパティ） |
| `@State` | その View 専用の「状態を覚えておく変数」。変わると画面が自動で再描画される |
| `@Binding` | 親から子 View に「状態への参照」を渡す仕組み（子が変更すると親にも反映される） |
| `@Observable` / `@StateObject` | 複数の View で共有する「データを持つクラス」に付ける印。中身が変わると関連する View が再描画される |
| NavigationStack | 画面遷移（次のページに進む）を管理する仕組み |
| Preview（プレビュー） | ビルド・実行しなくても Xcode 上で View 単体の見た目を確認できる機能 |

## Xcode / プロジェクト構成の用語

| 用語 | わかりやすい意味 |
|---|---|
| `.xcodeproj` | Xcode の「プロジェクトファイル」。アプリ全体の設計図 |
| Target（ターゲット） | 「何をビルドするか」の単位。アプリ本体、テスト、パッケージなどそれぞれがターゲット |
| Scheme（スキーム） | 「どのターゲットを、どう実行/テストするか」の設定セット。実行ボタンの左上で選ぶやつ |
| Swift Package (SPM) | コードを再利用可能な「部品（パッケージ）」として切り出す仕組み。このプロジェクトの `Packages/ForensicCore` がこれ |
| Package.swift | そのパッケージの設計図ファイル（依存関係や含まれるコードを定義） |
| モジュール | `import ForensicCore` のように、他の場所から読み込める単位 |
| `@main` | 「このアプリはここから始まる」という目印。`ForensicApp.swift` についている |

## テスト関連の用語

| 用語 | わかりやすい意味 |
|---|---|
| Unit Test（単体テスト） | 1つの関数やロジックが正しく動くかを自動でチェックするコード |
| XCTest / Swift Testing | Swift でテストを書くための標準的な仕組み |
| assert（アサート） | テストの中で「これはこうなるはず」と確認する命令 |
